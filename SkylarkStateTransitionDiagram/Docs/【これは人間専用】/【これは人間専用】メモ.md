# 【これは人間専用】メモ

わたしと Codex さんや、 Copilot さんなどの AI エージェントとやりとりするとき、  
状態遷移図がないとコミュニケーションが取れないと思ったんだぜ（＾～＾）  

そこで、わたしに入力しやすいカスタマイズがたくさんできるよう、  
状態遷移図を描く専用ツールを作ってもらおうと思ったんだぜ（＾～＾）  

フレームワークとして、 MonoGame を使って作ってほしいんだぜ（＾～＾）  


超良い感じの画面だぜ（＾▽＾）！
使い方説明書をマークダウンファイルで作ってほしいぜ（＾～＾）
リポジトリーのトップにある README ファイルはディレクトリーツリー案内板みたいなリンク集にして、
説明書を Docs フォルダー下に作ってくれだぜ（＾▽＾）！


リポジトリー直下の README.md なんだが、以下のようなセクションに分けて欲しいんだぜ（＾～＾）

* 状態遷移図を読み書きしたい人
* このツールを作りたい人


おー、これは嬉しい README だぜ（＾▽＾）！

じゃあ、ツールの使い方説明書なんだが、
スクリーンショットや、
アイコンのサムネイル画像みたいなものも入れてくれないかだぜ（＾～＾）？


良い感じだぜ（＾▽＾）！

次はノードに UTF-8 BOM無しエンコーディングで、
日本語のラベルを付けれるようにしてほしいんだぜ（＾～＾）
できるかだぜ（＾～＾）？


日本語入力、良い感じだぜ（＾▽＾）！

エッジ（遷移）も UTF-8 BOM無しエンコーディングで日本語ラベルを付けれるようにしてほしいんだぜ（＾～＾）
水平のエッジなら上下に、垂直のエッジなら左右にラベルを付けれるようにしてほしいんだぜ（＾～＾）！


エッジにラベルを付けれたぜ、素晴らしいぜ（＾▽＾）！  


エッジが重なって困るんだぜ（＾～＾）
そこで段階的に対応してほしいんだぜ（＾～＾）
まず、エッジの線の始点と終点は、ノードの輪郭の円上に接するようにしてほしいんだぜ（＾～＾）
そして、マウスをドラッグして、円上の接点を動かせるようにしてほしいんだぜ（＾～＾）


エッジの始端と終端をノードの輪郭の円上で動かせたぜ（＾▽＾）！　素晴らしいぜ（＾▽＾）！
