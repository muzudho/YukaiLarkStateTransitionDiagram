# 【これは人間専用】メモ

わたしと Codex さんや、 Copilot さんなどの AI エージェントとやりとりするとき、  
状態遷移図がないとコミュニケーションが取れないと思ったんだぜ（＾～＾）  

そこで、わたしに入力しやすいカスタマイズがたくさんできるよう、  
状態遷移図を描く専用ツールを作ってもらおうと思ったんだぜ（＾～＾）  

フレームワークとして、 MonoGame を使って作ってほしいんだぜ（＾～＾）  


超良い感じの画面だぜ（＾▽＾）！
使い方説明書をマークダウンファイルで作ってほしいぜ（＾～＾）
リポジトリーのトップにある README ファイルはディレクトリーツリー案内板みたいなリンク集にして、
説明書を Docs フォルダー下に作ってくれだぜ（＾▽＾）！


リポジトリー直下の README.md なんだが、以下のようなセクションに分けて欲しいんだぜ（＾～＾）

* 状態遷移図を読み書きしたい人
* このツールを作りたい人


おー、これは嬉しい README だぜ（＾▽＾）！

じゃあ、ツールの使い方説明書なんだが、
スクリーンショットや、
アイコンのサムネイル画像みたいなものも入れてくれないかだぜ（＾～＾）？


良い感じだぜ（＾▽＾）！

次はノードに UTF-8 BOM無しエンコーディングで、
日本語のラベルを付けれるようにしてほしいんだぜ（＾～＾）
できるかだぜ（＾～＾）？


日本語入力、良い感じだぜ（＾▽＾）！

エッジ（遷移）も UTF-8 BOM無しエンコーディングで日本語ラベルを付けれるようにしてほしいんだぜ（＾～＾）
水平のエッジなら上下に、垂直のエッジなら左右にラベルを付けれるようにしてほしいんだぜ（＾～＾）！


エッジにラベルを付けれたぜ、素晴らしいぜ（＾▽＾）！  


エッジが重なって困るんだぜ（＾～＾）
そこで段階的に対応してほしいんだぜ（＾～＾）
まず、エッジの線の始点と終点は、ノードの輪郭の円上に接するようにしてほしいんだぜ（＾～＾）
そして、マウスをドラッグして、円上の接点を動かせるようにしてほしいんだぜ（＾～＾）


エッジの始端と終端をノードの輪郭の円上で動かせたぜ（＾▽＾）！　素晴らしいぜ（＾▽＾）！


それだぜ（＾～＾）
エッジを３次スプライン曲線にすると、ノードを迂回するエッジとか描けて良いかと思うんだが、
何か他にも良い案持ってるかだぜ（＾～＾）？


おお～、エッジを３次ベジェ曲線にしてくれだぜ（＾▽＾）！


エッジが３次ベジェ曲線になったぜ（＾▽＾）！　素晴らしいぜ（＾▽＾）！


おっ、そうだな、自己ループも欲しいよな、再入ってやつだな（＾▽＾）
ノードの円上の接点１つから出て、同じとこに戻ってくる円状の線でいいかな（＾～＾）？
円の形をしていると、ノードと見間違えないようにする工夫が要るな（＾～＾）
楕円の方がいいのかな（＾～＾）？


おー、自己ループを楕円形の３次ベジェ曲線で描けたぜ（＾▽＾）！　素晴らしいぜ（＾▽＾）！


開始ノードと終了ノードも欲しいんで、ノードで作ってみたんだが、黒色が選べないんだな（＾～＾）
パレットでも出てくるようにするかだぜ（＾～＾）？
２４万色とか選べるのは逆に不便なんで、厳選された色だけ選べればいいんだけど（＾～＾）
もし色にこだわる人がいたら、あとで設定ファイルで色を指定するようにすればいいし（＾～＾）


開始、終了ノードを作れたぜ（＾▽＾）！　素晴らしいぜ（＾▽＾）！  


図が大きくなってきたときに、画面をドラッグしてスクロールできるようにしてほしいんだぜ（＾～＾）
できる（＾～＾）？


画面のドラッグでスクロールできるようになったぜ（＾▽＾）！　素晴らしいぜ（＾～＾）！


ノードのサイズをグリッド吸着して変更できるようにすると
開始ノードを小さくしたり、たくさんのテキストが入るノードを大きくしたりできて便利だと
思うんだが、どう思う（＾～＾）？


ノードは円でいいかな、かわいいしな（＾▽＾）
文字は食み出てもいいんじゃないか（＾▽＾）


グリッドがサイズを規定していると考えれば、
ノードは半径のグリッドの倍数でいいかな（＾～＾）？
自然数だと楽だろ（＾～＾）


半グリッド単位でノードのサイズを変更できるようになったぜ（＾▽＾）！　素晴らしいぜ（＾▽＾）！
