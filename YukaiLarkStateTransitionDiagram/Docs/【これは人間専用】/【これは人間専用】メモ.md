それは UX の文脈だと、だいたい次の名前で語られることが多いぜ。

Progressive Disclosure
	最初から全部の機能を見せず、利用者の理解や必要性に応じて段階的に機能を開示する設計。

Progressive Onboarding
	チュートリアルを最初にまとめて出すのではなく、実際に使う場面で少しずつ覚えてもらう設計。

Scaffolding
	教育・学習設計寄りの言葉。最初は補助を多く出し、熟練するにつれて補助を減らしていく仕組み。

Adaptive UI / Adaptive UX
	利用者の熟練度や行動履歴に応じて、UI や説明量を変える設計。

Skill Acquisition / Mastery Curve
	利用者が初心者から熟練者へ成長していく過程そのものを指す言葉。ゲーム UX や学習 UXでよく使う。

これは、アシスタント UI + アクション提案の設計にできそうだぜ。

名前としては：

- Contextual Action
- Suggested Action
- Quick Action
- Assistant Panel
- Command Suggestion

ユカイラークは、常に画面を占有する説明役ではなく、

表示状態の案：

- 表示：画面内に出て案内する

開始ノードがまだ無いときの例：

利用者の熟練度に合わせた入口：

- 初心者：ユカイラークの [開始ノード作成] を押す

- 上級者：N / T などのショートカットで操作する
