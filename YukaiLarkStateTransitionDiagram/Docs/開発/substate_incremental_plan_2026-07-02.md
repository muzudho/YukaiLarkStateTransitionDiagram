# サブステート段階導入 実装計画 2026-07-02

## 背景

Version 2.0.0 に向けて、通常ノードの中にさらに 1 つのダイアグラムを持てる「サブステート」を追加したい。

ただし Version 1.* で作った JSON を壊さないことを優先する。最初からサブステートを作り込まず、Web ブラウザーのタブのように複数のダイアグラムを持てる構造へ段階的に寄せる。

## 用語

- ダイアグラムファイル: 保存 JSON 全体。
- ダイアグラムインスタンス: ノード、遷移、アシスト抑制などを持つ 1 枚の作図面。
- ルートダイアグラム: Version 1.* の図に相当する最初のダイアグラムインスタンス。
- サブステート: 通常ノードにひも付くダイアグラムインスタンス。将来、通常ノードの中身として開く。
- グローバル遷移: 通常ノードから外へ伸びている遷移を、サブステート内のどのノードからでも発火できる遷移として扱う考え方。

## 基本方針

今のダイアグラムを「1 つのインスタンス」として扱えるようにする。

現在は `Game1` が `_nodes` と `_transitions` を直接持ち、保存時は `DiagramDocument.Data.Nodes` と `DiagramDocument.Data.Transitions` に入れている。この形を、まず `DiagramInstance` のような単位へ包む。

最初の段階では画面の見た目や保存結果を大きく変えない。内部だけを「現在開いている 1 枚のダイアグラムインスタンスを編集している」形に寄せる。

## 仕様メモ 2026-07-03

複数タブ保存と日本語タブ名の読み書きまで確認できたため、次は通常ノードとサブステートの関連付け仕様を固める。

- サブステートを持てるのは［通常ノード］だけにする。［開始マーク］と［終了マーク］はサブステートを持たない。
- 1 つの［通常ノード］が持てるサブステートは最大 1 個にする。
- 実装上は `DiagramNode` に `int? SubstateDiagramId` のような nullable な関連 ID を追加する。
- サブステートの中身は新しい専用型を作らず、既存の `DiagramInstance` をそのまま使う。
- 親子関係の正本は親側の `DiagramNode.SubstateDiagramId` に持たせる。`DiagramInstance` 側へ親ノード ID を重複保存するのは、必要になるまで避ける。
- サブステート未作成の［通常ノード］で「中へ入る」操作をしたときは、新しい `DiagramInstance` を作成して親ノードへ関連付ける。
- 既にサブステートを持つ［通常ノード］で「中へ入る」操作をしたときは、関連付いたダイアグラムへ切り替える。
- 親ノードから外へ伸びる遷移をサブステート側のグローバル遷移として見せる機能は、関連付け操作が安定してから別フェーズで扱う。

サブステートを持つ［通常ノード］の見た目は二重枠を第一案にする。アイコン表示は、画面上のアイコン密度が上がりすぎるため当面採用しない。

終了マークとの混同を避けるため、サブステート二重枠は次の条件で描き分ける。

- 対象は［通常ノード］だけ。［開始マーク］や［終了マーク］には出さない。
- 終了マークの二重線は白系の細い同心円として維持する。
- サブステート二重枠は、通常ノードの色・選択色・テーマ色に寄せた補助リングにする。
- 補助リングはノード塗りと外周線の雰囲気を壊さない範囲で、外周線の少し内側または外側に置く。
- ホバー時のウニョウニョ外周、選択中の強調、リサイズハンドルと重なっても意味が読めるようにする。

## 段階案

### Phase 1: 1 枚の図をインスタンス化する

目的は、既存機能をほぼそのまま保ちながら、図の状態を 1 オブジェクトにまとめること。

- `DiagramInstance` を追加する。
- `DiagramInstance` は `Id`, `Title`, `Nodes`, `Transitions`, `AssistSuppression` を持つ。
- `Game1` の `_nodes`, `_transitions`, `_assistSuppression` は、当面は現在インスタンスへの参照またはプロパティとして扱う。
- Undo/Redo は当面、現在インスタンス単位で今まで通り動かす。
- JSON の旧形式読み込みは、読み込んだ `Data` をルートダイアグラム 1 個へ変換する。
- 保存は互換性を優先し、しばらくは現在の `Data` 形式を維持する。

この段階ではタブ UI を出さない。内部設計だけを準備する。

### Phase 2: 保存形式に複数インスタンスを入れられるようにする

目的は、1 ファイル内に複数ダイアグラムを持てる保存形式を追加すること。

- `DiagramDocument` に `Diagrams` と `ActiveDiagramId` を追加する。
- 旧 `Data` は互換読み込み用として残す。
- 読み込み時、`Diagrams` があればそれを使う。なければ旧 `Data` からルートダイアグラムを 1 個作る。
- 保存時は Version 2.0.0 用として `Diagrams` を出すか、移行期間だけ `Data` も併記するかを決める。

互換性を強くするなら、移行期間は `Data` にアクティブなルートダイアグラムも書いておく。ただし Version 1.* が保存し直すと複数インスタンス情報は失われる可能性があるため、運用上は「Version 2 で開いたファイルは Version 1 へ戻さない」を説明する必要がある。

### Phase 3: タブ UI を追加する

目的は、ユーザーが複数のダイアグラムインスタンスを切り替えられるようにすること。

- 画面上部またはヘッダー下にタブバーを追加する。
- 最初は「ルート」「図 2」「図 3」のような独立した図として扱う。
- タブ切り替え時は、選択、編集中テキスト、ドラッグ中操作を解除する。
- カメラ位置とズームは、ダイアグラムインスタンスごとに持つか、アプリ共通にするかを決める。
- Undo/Redo は当面、アクティブなタブ内だけに閉じるのが分かりやすい。

この段階でも、まだ通常ノードとの親子関係は作らない。

### Phase 4: 通常ノードとサブステートを関連付ける

目的は、通常ノードから中身のダイアグラムへ入れるようにすること。

- `DiagramNode` に `SubstateDiagramId` のような関連 ID を追加する。
- `SubstateDiagramId` は通常ノードだけが使う。開始マーク、終了マークでは無視する。
- 1 ノードにつき関連できるサブステートダイアグラムは 1 個にする。
- 通常ノードを選択して「中へ入る」操作で、対応するダイアグラムインスタンスを開く。
- サブステート未作成なら、新しいダイアグラムインスタンスを作るか確認する。
- タブ表示では、親ノード名をタブ名に使えるようにする。
- ルートへ戻る、親へ戻る、タブで切り替える、の操作を整理する。
- サブステートを持つ通常ノードは二重枠で表示する。終了マークの二重線と混同しないよう、通常ノード専用の補助リングとして描く。

### Phase 5: グローバル遷移を表現する

目的は、親の通常ノードから外へ出ている遷移を、サブステート内のどこからでも使えるイベントとして見せること。

- 親ノードから外へ出る遷移を、サブステート側では読み取り専用のグローバル遷移として表示する。
- 最初はサブステート画面の余白やインスペクターに「親から外へ出る遷移」として一覧するだけでもよい。
- 将来、サブステート内の任意ノードからその遷移へつながっているような視覚表現を追加する。
- 親側の遷移ラベル変更は、サブステート側の表示へ反映する。

## 保存形式案

将来の Version 2 系では、概念的には次の形に寄せる。

```json
{
  "FormatVersion": 3,
  "ActiveDiagramId": "root",
  "Diagrams": [
    {
      "Id": "root",
      "Title": "ルート",
      "Data": {
        "Nodes": [],
        "Transitions": []
      },
      "AssistSuppression": {
        "SuppressedSuggestions": []
      }
    }
  ]
}
```

既存 Version 1.* / 現行 Version 2 の `Data.Nodes` と `Data.Transitions` は、読み込み時にルートダイアグラムへ包む。

## 実装上の注意

- ノード ID は、ひとまずダイアグラムインスタンス内で一意ならよい。
- 将来、親子間参照やグローバル遷移を扱うなら、ダイアグラム ID とノード ID の組で参照できる設計にする。
- `NextNodeId` は保存せず、今まで通り読み込み後に最大 ID から計算してよい。
- 選択中ノード、選択中遷移、編集中テキスト、ドラッグ中状態は保存データではなくエディター状態として扱う。
- タブ切り替え時は編集中 IME を終了し、ドラッグやリンク作成中の状態をクリアする。
- アシスト抑制は、ユーザー体験としてはダイアグラムインスタンスごとに持つのが自然。

## 最初に着手するなら

最小の第一歩は、`DiagramNode.SubstateDiagramId` を追加し、JSON 保存、読み込み、Undo/Redo の複製で値が保持されるようにすること。

ただし UI と保存結果を同時に変えると不具合の切り分けが難しい。最初のパッチでは、サブステート作成操作までは入れず、既存 JSON 互換を保ったまま nullable ID が保存できる土台を作るのがよい。
